Example 1: Media Queries with Width

How it works:

This approach uses screen width to assume hover capability:

  • Desktop (≥769px): Assumes hover support
  • Mobile (≤768px): Assumes no hover support

Problems with this approach:

  • Small laptops support hover but may be ≤768px
  • Large tablets don't support hover but may be ≥769px
  • Screen size ≠ input method

Interactive Demo

Try hovering or tapping the button below:

Current behavior based on screen width:

CSS Code:

/* Assume desktop supports hover */
@media (min-width: 769px) {
  .demo-button:hover {
    background-color: #2980b9;
    transform: translateY(-2px);
  }
}

/* Assume mobile doesn't support hover */
@media (max-width: 768px) {
  .demo-button:active {
    background-color: #2980b9;
    transform: translateY(-2px);
  }
}

Example 3: User-Agent Sniffing

How it works:

This approach examines the user-agent string to determine device type:

  • Mobile keywords found: Assumes no hover support
  • No mobile keywords: Assumes hover support

Problems with this approach:

  • User-agent strings can be spoofed or modified
  • New devices might not match existing patterns
  • Desktop browsers can emulate mobile user-agents
  • Very fragile and unreliable

Interactive Demo

Try hovering or tapping the button below:

Detected device type based on User-Agent:

Detection Details:

Your User-Agent:

Regex Pattern: /Mobi|Android|iPhone|iPad/i

Pattern Match:

JavaScript Code:

const isTouchDevice = /Mobi|Android|iPhone|iPad/i.test(navigator.userAgent);

if (isTouchDevice) {
  document.body.classList.add("no-hover");
} else {
  document.body.classList.add("hover");
}

CSS Code:

body.hover .demo-button:hover {
  background-color: #2980b9;
  transform: translateY(-2px);
}

body.no-hover .demo-button:active {
  background-color: #2980b9;
  transform: translateY(-2px);
}

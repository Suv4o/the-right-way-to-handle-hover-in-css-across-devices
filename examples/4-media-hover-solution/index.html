Example 4: @media (hover) Solution ✓

How it works:

This modern CSS feature directly queries the device's hover capability:

  • @media (hover: hover): Device supports hover
  • @media (hover: none): Device doesn't support hover

Why this is the best approach:

  • ✅ Directly asks the device about hover capability
  • ✅ No assumptions based on screen size or user-agent
  • ✅ Works correctly with hybrid devices
  • ✅ Future-proof and standards-based
  • ✅ No JavaScript required
  • ✅ Excellent browser support

Interactive Demo

Try hovering or tapping the buttons below:

Hover Card

This card will have hover effects on hover-capable devices and tap effects on touch devices.

Current hover capability:

Browser Support:

  • Chrome: 38+
  • Firefox: 64+
  • Safari: 9+
  • Edge: 12+
  • Opera: 25+

CSS Code:

/* Devices that support hover (e.g. mouse, trackpad) */
@media (hover: hover) {
  .demo-button:hover {
    background-color: #2980b9;
    transform: translateY(-2px);
    box-shadow: 0 4px 8px rgba(0,0,0,0.2);
  }
  
  .card:hover {
    transform: translateY(-4px);
    box-shadow: 0 8px 25px rgba(0,0,0,0.15);
  }
}

/* Devices that don't support hover (e.g. touchscreens) */
@media (hover: none) {
  .demo-button:active {
    background-color: #2980b9;
    transform: translateY(-2px);
    box-shadow: 0 4px 8px rgba(0,0,0,0.2);
  }
  
  .card:active {
    transform: translateY(-2px);
    box-shadow: 0 4px 15px rgba(0,0,0,0.1);
  }
}
